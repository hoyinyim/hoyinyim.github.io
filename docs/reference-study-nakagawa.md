## 實際觀察

Header 同時容納品牌、特集、商品分類、贈禮、讀物、搜尋、會員、登入及購物車；首屏後以公告、特集、推薦、衣食住與贈禮建立高密度資訊架構。分類頁把大類、子類、特集、讀物及服務入口集中呈現。特集詳頁雖然長達二萬像素以上，仍透過主題段落、產品組、說明與下一批入口維持節奏。

高密度並不等於每項都做成相同卡片：全寬特集、列表、分類樹、公告 Ledger、橫向商品列並存。手機改以獨立導覽及分段，不能把桌面所有功能硬塞同一列。

## 可轉化原理

- Header 搜尋及 Menu 分工，讓桌面只保留四個主要入口。
- Publications 以清楚分類、狀態、Filter、結果數量管理高密度資料。
- Credentials 與 CV 使用列表、時間線、索引，而非固定卡片網格。
- Search 索引初始載入、鍵盤導航與清楚空狀態。

## 刻意不採用

不採用購物車、會員、商品價格、電商推薦、紅黃促銷標籤、商品輪播、工藝品牌圖像或購物分類名稱。學術網站不模擬商店，也不把成果分成商品卡。
