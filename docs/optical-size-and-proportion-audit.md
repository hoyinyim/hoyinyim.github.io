## 驗收素材與方法

以 Windows Chrome／Chromium 模擬器逐頁開啟，保留 249 張完整頁面截圖：14 個 Viewport × 11 頁（154 張）、1440×900 的 100／110／125／150／175／200％ 縮放（66 張）、首頁 320–1920px 的 29 個寬度（含 740／760／780px）。Safari 與 Firefox 在此 Windows 工作區未有可用執行檔；因此不把 Chrome 結果誤稱為該兩瀏覽器的實機結果。

所有最終完整截圖與人工接觸表位於 `docs/qa/optical-scale-final/`：

- `full/`：154 張逐頁、逐尺寸全頁圖。
- `zoom/`：66 張縮放圖。
- `critical-widths/`：29 張首頁中間寬度圖。
- `contact-sheets/`：14 張、每張彙整同一尺寸的 11 頁全頁圖。
- `menu-state/`：1440×900 與 390×844 的開啟選單圖與量測紀錄。

我以肉眼檢查首屏焦點、H1／正文尺度、圖片與校徽光學大小、古文字重量、行列節奏、footer 收束及手機重新編排；水平溢出與小字量測只作為輔助，不作通過依據。

## 人工問題與已修正處

1. 首頁原本以近乎全螢幕高度放置 Hero，學術簡介落在過深的下一屏；現將高度限制在 31–44rem，並縮短拓撲與三張索引卡。
2. 著作、期刊、研討會與教學的頁首，原先巨型古文字與 62–72vh Hero 共同壓過資料入口；現收斂字形尺寸／透明度，並以 25–35rem 的內容型 Hero 取代。
3. 臺北市立大學橫式校徽與成大方形標記同設 62px 時，前者光學重量明顯不足；已改成保留原比例的 108×62px，手機為 84×48px。
4. Header 操作文字、品牌英文、Menu 工具列、Footer email 原有 11.5–14px 的情形；已提高至至少 16px，仍保持名稱、導覽、操作三層層級。
5. Menu 左側古文字舞台與主項原先相互競爭；現將舞台由約 486px 收為約 414px，主項在 1440px 時由約 84px 收為 72px，六項與輔助連結可在單一桌面高度理解。

## 參考站同尺寸比較

以 1440×900 依序查看 [hashi.co.jp](https://hashi.co.jp/)、[kaitaksha.com](https://kaitaksha.com/)、[AA Design Lab](https://aa-design-lab.com/rule)、[早稻田計畫站](https://prj-foodecon.w.waseda.jp/)。四者均採大量留白與少量高權重元素，但首屏常只放品牌或單一標題。本網站因此不直接模仿其空白比例，而採「姓名／研究定位／可行動連結」同屏的學術入口；調整後：

- H1 保留主要焦點，但不再與巨型字形同時競爭。
- 正文和書目維持比參考站更高的資訊密度，以適合學術查閱。
- 首屏留白仍有呼吸感，卻不延遲人物簡介、著作分類或文獻清單。
- Menu 的字與圖比例控制為「文字主、古文字輔」，避免成為純視覺海報。

## 結論

本輪不以 token、Lighthouse 或「沒有 overflow」取代肉眼判斷。完成修正後，11 頁在 14 個尺寸、6 個縮放及中間寬度巡查下，未再發現需要以 CSS 補丁處理的尺寸失衡。詳見問題明細、元件矩陣、焦點地圖與前後對照。
